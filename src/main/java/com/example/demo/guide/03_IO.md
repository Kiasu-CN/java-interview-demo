## 一、IO 体系结构 ⭐（必须能画出）

### 1.1 字节流（处理任意数据：文本、图片、视频...）

```
InputStream（抽象类，字节输入流）
├── FileInputStream          — 文件输入
├── BufferedInputStream      — 缓冲输入（装饰器）
├── ByteArrayInputStream     — 字节数组输入
├── ObjectInputStream        — 对象反序列化输入
├── DataInputStream          — 基本类型输入
├── FilterInputStream        — 过滤输入（装饰器基类）
└── PipedInputStream         — 管道输入（线程间通信）

OutputStream（抽象类，字节输出流）
├── FileOutputStream          — 文件输出
├── BufferedOutputStream      — 缓冲输出（装饰器）
├── ByteArrayOutputStream     — 字节数组输出
├── ObjectOutputStream        — 对象序列化输出
├── DataOutputStream          — 基本类型输出
├── PrintStream               — 格式化输出（System.out 就是 PrintStream）
└── FilterOutputStream        — 过滤输出
```

### 1.2 字符流（专门处理文本，解决编码问题）

```
Reader（抽象类，字符输入流）
├── FileReader               — 文件字符输入
├── BufferedReader            — 缓冲字符输入（有 readLine()）
├── InputStreamReader         — 字节→字符的桥梁（指定编码）
├── StringReader              — 字符串输入
├── CharArrayReader           — 字符数组输入
└── PipedReader               — 管道字符输入

Writer（抽象类，字符输出流）
├── FileWriter                — 文件字符输出
├── BufferedWriter             — 缓冲字符输出（有 newLine()）
├── OutputStreamWriter         — 字符→字节的桥梁（指定编码）
├── StringWriter               — 字符串输出
├── CharArrayWriter            — 字符数组输出
├── PrintWriter                — 格式化输出
└── PipedWriter                — 管道字符输出
```

### 1.3 核心记忆口诀

```
字节流结尾是 Stream，字符流结尾是 Reader/Writer
输入是 Input/Reader，输出是 Output/Writer
带 Buffered 的是缓冲流（性能优化）
InputStreamReader 是字节→字符的桥梁
```

## 三、字节流 ⭐

### 3.1 FileInputStream / FileOutputStream

```java
// ========== 文件复制（最基础的字节流操作） ==========
public static void copyFile(String src, String dest) throws IOException {
    try (FileInputStream fis = new FileInputStream(src);
         FileOutputStream fos = new FileOutputStream(dest)) {
        byte[] buffer = new byte[1024]; // 缓冲区
        int bytesRead;
        while ((bytesRead = fis.read(buffer)) != -1) {
            fos.write(buffer, 0, bytesRead); // 注意写入实际读取的字节数
        }
    }
}

// FileOutputStream 构造参数说明
new FileOutputStream("file.txt");       // 覆盖写入
new FileOutputStream("file.txt", true); // 追加写入（第二个参数 append=true）

// read 方法的三种形式
fis.read();                  // 读一个字节，返回 0~255，到末尾返回 -1
fis.read(byte[] buf);        // 读到 buf 中，返回实际读取的字节数
fis.read(byte[] buf, off, len); // 读到 buf[off] 开始的位置，最多读 len 个
```

### 3.2 BufferedInputStream / BufferedOutputStream

```java
// 缓冲流：内部维护 8KB 缓冲区，减少系统调用次数
// 性能对比：缓冲流比单字节读写快 10~100 倍！

public static void copyFileFast(String src, String dest) throws IOException {
    try (BufferedInputStream bis = new BufferedInputStream(new FileInputStream(src));
         BufferedOutputStream bos = new BufferedOutputStream(new FileOutputStream(dest))) {
        byte[] buffer = new byte[8192];
        int bytesRead;
        while ((bytesRead = bis.read(buffer)) != -1) {
            bos.write(buffer, 0, bytesRead);
        }
        bos.flush(); // 刷新缓冲区（try-with-resources 会自动调用 close → flush）
    }
}
```

### 3.3 ByteArrayInputStream / ByteArrayOutputStream

```java
// 内存中的字节流，不需要文件
// 常用于：测试、数据转换、合并字节数组
```

## 四、字符流 ⭐

### 4.1 FileReader / FileWriter

```java
// 字符流：每次读一个字符（2字节），适合处理中文
// FileWriter 内部使用了 StreamEncoder，默认 UTF-8

// 写文件
try (FileWriter fw = new FileWriter("output.txt")) {
    fw.write("Hello 你好");
    fw.write("\r\n");           // 换行（Windows \r\n, Linux \n）
    fw.append("追加内容");
    fw.flush();
}

// 读文件
try (FileReader fr = new FileReader("input.txt")) {
    char[] buffer = new char[1024];
    int charsRead;
    while ((charsRead = fr.read(buffer)) != -1) {
        System.out.print(new String(buffer, 0, charsRead));
    }
}

// 面试题：字节流 vs 字符流的区别？
// 1. 字节流操作 byte（8bit），字符流操作 char（16bit）
// 2. 字节流可以处理任何数据，字符流只能处理文本
// 3. 字符流内部有编码转换（字符 ←→ 字节），处理中文不会乱码
// 4. 字符流 = 字节流 + 编码表
```

### 4.2 BufferedReader / BufferedWriter

```java
// BufferedReader 有 readLine() 方法！逐行读取最常用
try (BufferedReader br = new BufferedReader(new FileReader("input.txt"));
     BufferedWriter bw = new BufferedWriter(new FileWriter("output.txt"))) {

    String line;
    while ((line = br.readLine()) != null) {  // readLine() 不包含换行符
        System.out.println(line);
        bw.write(line);
        bw.newLine();  // 跨平台的换行符
    }
}
```

### 4.3 InputStreamReader / OutputStreamWriter（桥梁）⭐

```java
// 字节流 → 字符流的桥梁，可以指定编码
// 这是解决中文乱码的关键！

// 用指定编码读取文件
try (BufferedReader br = new BufferedReader(
        new InputStreamReader(
            new FileInputStream("gbk.txt"), "GBK"))) {  // 指定GBK编码读取
    String line;
    while ((line = br.readLine()) != null) {
        System.out.println(line);
    }
}

// 用指定编码写入文件
try (BufferedWriter bw = new BufferedWriter(
        new OutputStreamWriter(
            new FileOutputStream("utf8.txt"), StandardCharsets.UTF_8))) {
    bw.write("你好世界");
}

// 面试题：如何解决中文乱码？
// 答：编码和解码使用相同的字符集。
// 读取时用 InputStreamReader 指定源文件编码
// 写入时用 OutputStreamWriter 指定目标编码
```

## 五、编码与乱码 ⭐

```java
// 常见编码：
// ASCII    — 1字节，英文
// GBK      — 2字节，中文（国标）
// UTF-8    — 1~4字节变长，国际标准
// ISO-8859-1 — 1字节，西欧语言

// 编码：String → byte[]
String s = "你好";
byte[] gbk = s.getBytes("GBK");        // [0xC4, 0xE3, 0xBA, 0xC3] → 4字节
byte[] utf8 = s.getBytes("UTF-8");      // [0xE4, 0xBD, 0xA0, 0xE5, 0xA5, 0xBD] → 6字节

// 解码：byte[] → String
String s1 = new String(gbk, "GBK");     // "你好"
String s2 = new String(gbk, "UTF-8");   // "ÄãºÃ" → 乱码！编码解码不一致

// 乱码的两种情况：
// 1. 编码用UTF-8，解码用GBK → 乱码
// 2. 用ISO-8859-1读取中文 → 每个中文字符被拆成问号

// 解决乱码的万能方法：
// 先用错误编码解码回 byte[]，再用正确编码重新解码
String wrong = new String(utf8, "GBK");         // 乱码
byte[] raw = wrong.getBytes("GBK");             // 还原回字节
String right = new String(raw, "UTF-8");        // 正确解码 → "你好"
```

## 六、NIO（New IO）⭐

### 6.1 BIO vs NIO vs AIO

| | BIO | NIO | AIO |
|--|-----|-----|-----|
| 全称 | Blocking I/O | Non-blocking I/O | Asynchronous I/O |
| 模型 | 同步阻塞 | 同步非阻塞 | 异步非阻塞 |
| 核心 | Stream（单向） | Channel + Buffer + Selector | CompletionHandler |
| 阻塞 | 读/写时线程阻塞 | read返回0表示无数据 | 操作系统完成后回调 |
| 连接处理 | 一连接一线程 | 一线程处理多连接 | 由OS完成后通知 |
| 适用场景 | 连接少且固定 | 连接多且活跃度低 | 连接多且活跃度高 |

### 6.2 NIO 三大核心组件

**Buffer（缓冲区）：**

```java
// Buffer 是一块可读写的内存块
// 子类：ByteBuffer, CharBuffer, IntBuffer, LongBuffer...

ByteBuffer buf = ByteBuffer.allocate(1024);     // 堆内缓冲区
ByteBuffer directBuf = ByteBuffer.allocateDirect(1024); // 直接缓冲区（堆外内存，零拷贝）

// 四个核心属性：
// capacity — 容量（不变）
// position — 当前读写位置
// limit    — 读写上限
// mark     — 标记位置

// 写数据 → 读数据 的标准流程：
buf.put("Hello".getBytes());  // 写入数据，position 移动
buf.flip();                    // 切换为读模式：limit=position, position=0
while (buf.hasRemaining()) {
    System.out.print((char) buf.get()); // 读取数据
}
buf.clear();                   // 清空缓冲区：position=0, limit=capacity（数据未删除）

// 其他常用方法
buf.rewind();     // position=0，可重新读
buf.compact();    // 将未读数据移到开头，position 指向未读数据之后
buf.mark();       // 标记当前 position
buf.reset();      // 恢复到 mark 的位置
```

**Channel（通道）：**

```java
// Channel 是双向的（可读可写），Stream 是单向的
// 常用 Channel：FileChannel, SocketChannel, ServerSocketChannel, DatagramChannel

// FileChannel 读写文件
try (FileChannel inChannel = new FileInputStream("src.txt").getChannel();
     FileChannel outChannel = new FileOutputStream("dest.txt").getChannel()) {

    ByteBuffer buf = ByteBuffer.allocate(1024);
    while (inChannel.read(buf) != -1) {
        buf.flip();
        outChannel.write(buf);
        buf.clear();
    }

    // 零拷贝 transferTo（直接在内核空间传输，不经过用户空间）
    inChannel.transferTo(0, inChannel.size(), outChannel);
}
```

**Selector（选择器/多路复用器）：**

```java
// Selector 让一个线程管理多个 Channel（网络编程核心）
// 只能用于网络 Channel（SocketChannel），不能用于 FileChannel

Selector selector = Selector.open();
ServerSocketChannel serverChannel = ServerSocketChannel.open();
serverChannel.bind(new InetSocketAddress(8080));
serverChannel.configureBlocking(false); // 必须设为非阻塞
serverChannel.register(selector, SelectionKey.OP_ACCEPT); // 注册感兴趣的事件

// 事件类型：
// OP_ACCEPT  — 接受连接
// OP_CONNECT — 连接就绪
// OP_READ    — 读就绪
// OP_WRITE   — 写就绪

while (true) {
    selector.select(); // 阻塞，直到有事件发生
    Set<SelectionKey> keys = selector.selectedKeys();
    Iterator<SelectionKey> it = keys.iterator();
    while (it.hasNext()) {
        SelectionKey key = it.next();
        if (key.isAcceptable()) {
            // 处理新连接
            SocketChannel sc = serverChannel.accept();
            sc.configureBlocking(false);
            sc.register(selector, SelectionKey.OP_READ);
        } else if (key.isReadable()) {
            // 处理读事件
            SocketChannel sc = (SocketChannel) key.channel();
            ByteBuffer buf = ByteBuffer.allocate(1024);
            sc.read(buf);
        }
        it.remove(); // 必须手动移除已处理的事件
    }
}
```

## 八、序列化 ⭐

### 8.1 基本用法

```java
// Serializable — 标记接口（无方法），表示该类可被序列化
class User implements Serializable {
    // serialVersionUID：版本控制标识
    // 如果不指定，JVM 会自动生成（基于类结构），类结构变化后反序列化会失败
    private static final long serialVersionUID = 1L;

    String name;
    int age;
    transient String password;  // transient 修饰的字段不参与序列化
    static String country = "CN"; // static 字段不参与序列化

    User(String name, int age, String password) {
        this.name = name;
        this.age = age;
        this.password = password;
    }

    @Override
    public String toString() {
        return "User{name='" + name + "', age=" + age
             + ", password='" + password + "'}";
    }
}

// 序列化：对象 → 文件
public static void serialize(Object obj, String filePath) throws IOException {
    try (ObjectOutputStream oos = new ObjectOutputStream(
            new FileOutputStream(filePath))) {
        oos.writeObject(obj);
    }
}

// 反序列化：文件 → 对象
public static Object deserialize(String filePath)
        throws IOException, ClassNotFoundException {
    try (ObjectInputStream ois = new ObjectInputStream(
            new FileInputStream(filePath))) {
        return ois.readObject();
    }
}

// 使用示例
User user = new User("张三", 25, "123456");
serialize(user, "user.dat");

User restored = (User) deserialize("user.dat");
System.out.println(restored);
// User{name='张三', age=25, password='null'}  ← password 是 transient，为 null
```

### 8.2 序列化面试题

```
Q1: serialVersionUID 有什么作用？
A: 版本控制。反序列化时 JVM 对比 serialVersionUID，不一致则抛
   InvalidClassException。建议显式指定，避免类结构变化导致反序列化失败。

Q2: transient 关键字的作用？
A: 修饰的字段不参与序列化。反序列化后为默认值（引用类型 null，基本类型 0/false）。

Q3: static 变量能被序列化吗？
A: 不能。序列化是针对对象实例的，static 属于类级别。

Q4: 序列化能保存集合吗？
A: 可以，但集合中的每个元素也必须实现 Serializable。

Q5: 如果成员变量是另一个对象，该对象的类也必须实现 Serializable 吗？
A: 是的，否则会抛 NotSerializableException。

Q6: 如何实现深拷贝？
A: 将对象序列化到 ByteArrayOutputStream，再反序列化回来。
   这是一种通用的深拷贝方法，但性能较差。
```

```java
// 通过序列化实现深拷贝
@SuppressWarnings("unchecked")
public static <T extends Serializable> T deepCopy(T obj) throws IOException, ClassNotFoundException {
    ByteArrayOutputStream baos = new ByteArrayOutputStream();
    try (ObjectOutputStream oos = new ObjectOutputStream(baos)) {
        oos.writeObject(obj);
    }
    try (ObjectInputStream ois = new ObjectInputStream(
            new ByteArrayInputStream(baos.toByteArray()))) {
        return (T) ois.readObject();
    }
}
```

## 十、常见面试题汇总

### 10.1 基础概念题

**Q1: 字节流和字符流有什么区别？**
```
1. 操作单位不同：字节流操作 byte（8位），字符流操作 char（16位）
2. 使用场景：字节流可处理任意数据，字符流专门处理文本
3. 编码处理：字符流内置编码转换，字节流没有
4. 核心类：字节流 InputStream/OutputStream，字符流 Reader/Writer
5. 缓冲：字节流用 BufferedXXStream，字符流用 BufferedXXer
```

**Q2: BIO、NIO、AIO 的区别？**
```
BIO（同步阻塞）：
  - 一连接一线程
  - 读写操作阻塞线程
  - 适用于连接数少且固定的场景

NIO（同步非阻塞）：
  - Channel + Buffer + Selector
  - 一个线程可管理多个连接
  - 适用于连接数多但活跃度低的场景（如聊天服务器）

AIO（异步非阻塞）：
  - 操作系统完成 IO 操作后回调通知
  - 适用于连接数多且活跃度高的场景
```

**Q3: InputStream 里的 read() 方法返回值为什么是 int 而不是 byte？**
```
因为 byte 范围是 -128~127，而 read() 到达末尾返回 -1。
如果返回 byte 类型，文件内容中的 -1 和末尾标识 -1 无法区分。
所以用 int（0~255 表示有效字节，-1 表示末尾）。
```

**Q4: 为什么要用缓冲流？**
```
不带缓冲的流：每读/写一个字节就进行一次系统调用 → 慢
带缓冲的流：先读写到内存缓冲区（默认8KB），满了再一次性系统调用 → 快
性能差距可达 10~100 倍。
```

**Q5: 装饰器模式在 IO 中的应用？**
```
Java IO 大量使用装饰器模式：
  new BufferedReader(new InputStreamReader(new FileInputStream("file.txt"), "UTF-8"))

  FileInputStream       → 基础组件（读取字节）
  InputStreamReader     → 装饰器（字节→字符）
  BufferedReader        → 装饰器（添加缓冲和 readLine）

特点：可以灵活组合，层层包装增强功能。
```

**Q6: 统计一个文本文件中每个单词出现的频率，按频率降序输出**
```java
public static void wordFrequency(String filePath) throws IOException {
    Map<String, Long> freq = new HashMap<>();
    try (BufferedReader br = new BufferedReader(new FileReader(filePath))) {
        String line;
        while ((line = br.readLine()) != null) {
            String[] words = line.toLowerCase().split("\\W+");
            for (String word : words) {
                if (!word.isEmpty()) {
                    freq.merge(word, 1L, Long::sum);
                }
            }
        }
    }
    // 按频率降序排序
    freq.entrySet().stream()
        .sorted(Map.Entry.<String, Long>comparingByValue().reversed())
        .forEach(e -> System.out.println(e.getKey() + ": " + e.getValue()));
}

// Stream 版本（更简洁）
public static void wordFrequencyStream(String filePath) throws IOException {
    Files.lines(Paths.get(filePath))
        .flatMap(line -> Arrays.stream(line.toLowerCase().split("\\W+")))
        .filter(w -> !w.isEmpty())
        .collect(Collectors.groupingBy(w -> w, Collectors.counting()))
        .entrySet().stream()
        .sorted(Map.Entry.<String, Long>comparingByValue().reversed())
        .forEach(e -> System.out.println(e.getKey() + ": " + e.getValue()));
}
```

```java
public static void copyDirectory(File src, File dest) throws IOException {
    if (src.isDirectory()) {
        File newDir = new File(dest, src.getName());
        newDir.mkdirs();
        File[] children = src.listFiles();
        if (children != null) {
            for (File child : children) {
                copyDirectory(child, newDir);
            }
        }
    } else {
        Files.copy(src.toPath(), new File(dest, src.getName()).toPath(),
                   StandardCopyOption.REPLACE_EXISTING);
    }
}
```

**Q9: 实现一个简单的日志写入器**
```java
class SimpleLogger {
    private final PrintWriter writer;

    SimpleLogger(String filePath) throws IOException {
        this.writer = new PrintWriter(
            new BufferedWriter(new FileWriter(filePath, true)), // 追加模式
            true);  // autoFlush=true
    }

    void info(String msg) {
        writer.println(String.format("[%s][INFO] %s",
            LocalDateTime.now().format(DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss")),
            msg));
    }

    void error(String msg, Throwable t) {
        writer.printf("[%s][ERROR] %s%n",
            LocalDateTime.now().format(DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss")),
            msg);
        t.printStackTrace(writer);
    }

    void close() { writer.close(); }
}
```
