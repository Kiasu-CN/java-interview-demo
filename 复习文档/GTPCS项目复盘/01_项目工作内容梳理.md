# GTPCS 项目工作内容梳理

## 1. 项目定位

**油气集输生产运营平台（GTPCS）** 本质上不是单纯的后台管理系统，而是一套围绕 **生产数据采集同步、异常治理、业务运营、值班协同、报表看板** 的综合业务平台。

从代码结构看，它是：

- 前后端分仓：`gtpcs-backend` + `gtpcs-frontend`
- 后端多模块 Maven 工程，不是典型 Spring Cloud 微服务拆分
- 你这段实习最相关的核心模块主要集中在：
  - `gtpcs-backend/gtpcs-yycy`
  - `gtpcs-backend/gtpcs-yycy-job`
- 前端对应页面主要在：
  - `gtpcs-frontend/src/views/yycy/*`

更稳妥的求职口径可以概括为：

**我做的是面向生产运营场景的后端开发，不只是写 CRUD，而是同时覆盖了数据同步、定时任务、规则 SQL、消息通知、报表查询和前后端联动。**

## 2. 项目整体业务闭环

按代码链路看，这个项目大致是下面这个闭环：

1. 外部生产数据进入系统  
   来自数据湖接口、站库业务表、时序数据表等。

2. 后端执行数据同步、落库与汇总  
   典型代码在 `gtpcs-yycy-job` 的各类 `*Task`、`*ServiceImpl`。

3. 基于规则做异常检查和质量考核  
   将异常记录落到统一异常表，再做明细查询、考核评分、人工录入补充。

4. 通过消息、值班、夜查等模块推动业务动作  
   包括站内消息、石化通推送、排班提醒、夜查通知等。

5. 对外提供报表、晨报、看板、导出能力  
   面向业务人员做统计展示、过程跟踪和管理闭环。

这也是这套项目最适合校招后端去讲的地方：  
**它的主线是“数据处理 + 规则校验 + 任务调度 + 消息触达 + 结果展示”，不是单点页面开发。**

## 3. 主线一：异常检查与质量考核闭环

### 这条线主要在做什么

把生产系统里的异常数据、未审核数据、未按时录入数据、人员健康缺失数据、报警率等问题，统一整理成异常记录，再提供明细查询、统计评分和人工补录能力。

### 核心代码文件

- 定时任务入口  
  `gtpcs-backend/gtpcs-yycy-job/src/main/java/cn/com/victorysoft/pcs/job/Task/YcsjjcTask.java`
- 异常检查核心服务  
  `gtpcs-backend/gtpcs-yycy-job/src/main/java/cn/com/victorysoft/pcs/job/service/impl/YcsjjcServiceImpl.java`
- 手动触发入口  
  `gtpcs-backend/gtpcs-yycy-job/src/main/java/cn/com/victorysoft/pcs/job/controller/YcsjjcController.java`
- 规则 SQL / merge 入库  
  `gtpcs-backend/gtpcs-yycy-job/src/main/resources/config/mappers/job/YcsjjcMapper.xml`
- 质量考核页面接口  
  `gtpcs-backend/gtpcs-yycy/src/main/java/cn/com/victorysoft/pcs/zlkh/controller/ZlkhController.java`
- 质量考核服务  
  `gtpcs-backend/gtpcs-yycy/src/main/java/cn/com/victorysoft/pcs/zlkh/service/impl/ZlkhServiceImpl.java`
- 评分与明细 SQL  
  `gtpcs-backend/gtpcs-yycy/src/main/resources/config/mappers/zlkh/ZlkhMapper.xml`
- 前端页面  
  `gtpcs-frontend/src/views/yycy/zlkh/index.vue`  
  `gtpcs-frontend/src/views/yycy/zlkh/input.vue`

### 从代码上能看出的核心动作

- `YcsjjcTask` 按天定时执行异常检查与人员健康检查。
- `YcsjjcServiceImpl` 不是只做单一查询，而是把异常拆成多类链路：
  - 基础异常规则
  - 实时数据转储率
  - 输差审核未审
  - 值班记录未填
  - 轻烃产销未采集
  - 报警处置率 / 确认率
  - 在用设备缺少运行记录
- `YcsjjcMapper.xml` 里有大量规则型 SQL，并通过 `MERGE INTO ssc_epbp_zljcycjl` 做幂等入库。
- `ZlkhMapper.xml` 基于异常记录做明细分页、日/月/年统计和单位评分，里面用到了较完整的 `CTE + 权重模板 + 聚合评分`。
- `ZlkhController` 还支持人工补录异常，说明这条链路不仅有自动检查，也有业务兜底。

### 这条线体现的后端能力

- 规则抽象能力
- 复杂 SQL / Mapper 维护能力
- 定时任务 + 批处理能力
- 幂等写入能力
- 接口 + 页面联动交付能力

### 最值得写进简历的内容

- 基于定时任务和规则 SQL 搭建异常检查链路
- 将多类异常统一落表并支撑后续查询、评分和补录
- 做过 `MERGE` 式幂等入库，而不是只做查询
- 做过日 / 月 / 年维度质量考核统计

### 面试时怎么讲

建议这样讲：

1. 先说这不是简单列表查询，而是一套异常治理闭环。
2. 再说你负责的是“任务检查 -> 异常落表 -> 明细查询 -> 评分统计 -> 人工补录”。
3. 最后强调你做了规则 SQL、批量入库、评分口径这些偏后端的事情。

## 4. 主线二：定时任务、数据同步与汇总链路

### 这条线主要在做什么

把数据湖、业务系统、时序库中的生产数据按时间周期拉取、同步、入库、汇总，支撑后续的报表、异常检查和看板展示。

### 核心代码文件

- 储气库同步任务总入口  
  `gtpcs-backend/gtpcs-yycy-job/src/main/java/cn/com/victorysoft/pcs/job/Task/CqkDataTask.java`
- 手动触发同步接口  
  `gtpcs-backend/gtpcs-yycy-job/src/main/java/cn/com/victorysoft/pcs/job/controller/CqkDataController.java`
- 典型同步实现
  - `.../service/impl/CqkCqjrsjServiceImpl.java`
  - `.../service/impl/CqkByxjlServiceImpl.java`
  - `.../service/impl/CqkTrqlljyxjlServiceImpl.java`
  - `.../service/impl/CqkZksbbgztServiceImpl.java`
- 当前表 / 班次汇总任务入口  
  `gtpcs-backend/gtpcs-yycy-job/src/main/java/cn/com/victorysoft/pcs/job/Task/SjhzTask.java`
- 典型汇总实现
  - `.../service/impl/LljServiceImpl.java`
  - `.../service/impl/PzjzServiceImpl.java`
  - `.../service/impl/GuanServiceImpl.java`

### 从代码上能看出的核心动作

- `CqkDataTask` 负责按时间调度多类同步任务，如采气井、注气井、监测井、排液井、泵运行记录、天然气流量计、站库用电、设备变更状态等。
- `Cqk*ServiceImpl` 普遍采用统一模式：
  - 拼装接口参数
  - 调数据湖接口 `DataLakeApiUtil.doGet`
  - 分页拉取
  - 解析 JSON
  - 按批次入库
- 这些实现里多次出现“按 KingBase 参数上限控制批量大小”的处理，说明你做过数据库约束下的工程化落库。
- `SjhzTask` 负责大量设备“当前表 / 班次表”汇总任务。
- `LljServiceImpl`、`PzjzServiceImpl` 这类服务会按设备清单取数，再做时间段聚合、读差计算、班次落库。
- `GuanServiceImpl`、`JckServiceImpl` 等实现里还能看到 `taosJdbcTemplate`，说明这套链路不只是关系库查询，还涉及时序库落数。

### 这条线体现的后端能力

- 定时任务开发能力
- 外部接口数据接入能力
- 分页拉取与批量入库能力
- 多数据源 / 跨库处理能力
- 面向生产数据的批处理能力

### 最值得写进简历的内容

- 负责多类定时任务与数据同步能力开发
- 处理过“分页拉取 + 脏数据过滤 + 分批入库”的完整链路
- 做过按数据库参数限制控制批量写入
- 做过生产数据的当前表 / 班次表汇总

### 面试时怎么讲

建议突出这几点：

1. 你做的不只是 CRUD，而是“数据拉取 + 清洗 + 分页 + 批量落库”。
2. 你考虑过 KingBase 参数限制，所以做了分批写入。
3. 你接触过关系库和时序库，不只是单一 MySQL 风格开发。

## 5. 主线三：消息推送、排班提醒与变更即时通知

### 这条线主要在做什么

围绕消息中心，把排班提醒、生产概况、原油零销计划变更等事件，转成站内消息和石化通通知，并管理推送状态。

### 核心代码文件

- 消息管理接口  
  `gtpcs-backend/gtpcs-yycy/src/main/java/cn/com/victorysoft/pcs/xxts/controller/XxglController.java`
- 消息服务  
  `gtpcs-backend/gtpcs-yycy/src/main/java/cn/com/victorysoft/pcs/xxts/service/impl/XxglServiceImpl.java`
- 消息 SQL  
  `gtpcs-backend/gtpcs-yycy/src/main/resources/config/mappers/xxts/XxglMapper.xml`
- 定时任务与动态调度  
  `gtpcs-backend/gtpcs-yycy/src/main/java/cn/com/victorysoft/pcs/xxts/task/PbxxTask.java`
- 石化通推送  
  `gtpcs-backend/gtpcs-yycy/src/main/java/cn/com/victorysoft/pcs/xxts/controller/ShtMessagePush.java`
- 前端页面  
  `gtpcs-frontend/src/views/yycy/xxts/xxcx.vue`  
  `gtpcs-frontend/src/views/yycy/xxts/xxtssz.vue`

### 从代码上能看出的核心动作

- `XxglController` 提供消息新增、分页查询、更新、统计等接口。
- `XxglMapper.xml` 区分了 `ZBXX` 与 `SCGK` 的接收人查询方式，说明消息模型并不完全统一。
- `PbxxTask` 自己维护 `ThreadPoolTaskScheduler`，从数据库表里读取推送时间配置并动态刷新任务。
- `executeInsertPbxx` 会按排班记录生成值班消息。
- `executeInsertYywxjh` 会读取变更日志、按业务键去重、精准查当前计划，再给配置的接收人逐人写消息。
- `executePbxxShtSendByTime` 会按“类型 + 时间窗口 + tszt”精准推送，并在成功后更新推送状态。
- `ShtMessagePush` 负责调用外部石化通接口完成触达。

### 这条线体现的后端能力

- 任务调度能力
- 消息链路设计能力
- 状态流转能力
- 外部接口集成能力
- 运维友好能力（手动触发 / 自检 / 刷新）

### 最值得写进简历的内容

- 动态调度，而不是把时间写死在代码里
- 按人单独生成和推送消息，而不是共享一条消息
- 推送状态 `tszt` 管理
- 计划变更触发消息生成与精准推送

### 面试时怎么讲

建议主打：

1. 这不是“发通知”，而是完整的消息链路。
2. 你做了动态任务刷新、消息落库、外部推送、状态更新。
3. 你能解释为什么要“按人单独推送”，以及如何避免重复推送。

## 6. 主线四：报表、晨报与导出能力

### 这条线主要在做什么

把生产数据按业务口径整理成报表、晨报和可导出的结构化结果，面向业务侧做统计和展示。

### 核心代码文件

- 报表接口  
  `gtpcs-backend/gtpcs-yycy/src/main/java/cn/com/victorysoft/pcs/report/controller/ReportController3.java`
- 晨报聚合  
  `gtpcs-backend/gtpcs-yycy/src/main/java/cn/com/victorysoft/pcs/report/service/impl/SccbServiceImpl.java`
- 原油零销报表  
  `gtpcs-backend/gtpcs-yycy/src/main/java/cn/com/victorysoft/pcs/report/service/impl/YylxServiceImpl.java`
- 典型 SQL  
  `gtpcs-backend/gtpcs-yycy/src/main/resources/config/mappers/report/YylxMapper.xml`
- 前端报表页面  
  `gtpcs-frontend/src/views/yycy/bb/*.vue`
- 导出工具  
  `gtpcs-frontend/static/js/excel/exportHtmlTableUtil.js`

### 从代码上能看出的核心动作

- `ReportController3` 下面不是一个报表，而是一组按不同业务维度拆分的报表接口。
- `SccbServiceImpl` 会把多个查询结果合并成晨报结构，并统一拼 KPI。
- `YylxMapper.xml` 明显做了业务口径处理：
  - 先构造目标维度
  - 再做日数据聚合
  - 再做月累计聚合
  - 最后补齐无数据维度
- 前端 `exportHtmlTableUtil.js` 不是简单导出字符串，而是处理了合并单元格、富文本、样式、背景色、标题格式。

### 这条线体现的后端能力

- 复杂查询与口径整理能力
- 多数据集聚合能力
- 报表接口设计能力
- 前后端协作交付能力

### 最值得写进简历的内容

- 做过业务报表、晨报聚合接口
- 做过按天 / 按月口径的聚合统计
- 做过导出能力与前端联动

### 面试时怎么讲

建议讲法：

1. 报表不是页面拼接，而是先统一业务口径。
2. 你做了“维度补齐、月累计、KPI 聚合”这类真正偏后端的事情。
3. 如果面试官继续问，你可以讲到导出和展示联动。

## 7. 主线五：生产运行会、值班夜查与数据库通知驱动推送

### 这条线主要在做什么

围绕值班夜查和生产运行会，既做业务记录的生成与维护，也做变更后的自动通知和推送闭环。

### 核心代码文件

- 夜查抽查接口  
  `gtpcs-backend/gtpcs-yycy/src/main/java/cn/com/victorysoft/pcs/zbxt/zbyc/controller/ZbycscController.java`
- 夜查随机逻辑  
  `gtpcs-backend/gtpcs-yycy/src/main/java/cn/com/victorysoft/pcs/zbxt/zbyc/service/impl/ZbycscServiceImpl.java`
- 夜查通知任务  
  `gtpcs-backend/gtpcs-yycy/src/main/java/cn/com/victorysoft/pcs/zbxt/zbyc/task/ZbycNotifyTask.java`
- 数据库通知监听  
  `gtpcs-backend/gtpcs-yycy/src/main/java/cn/com/victorysoft/pcs/zbxt/zbyc/task/ZbycChangeNotifyListener.java`
- 触发器 / 通知 SQL  
  `gtpcs-backend/gtpcs-yycy/src/main/resources/config/mappers/zbxt/ZbycTaskMapper.xml`
- 值班记录接口  
  `gtpcs-backend/gtpcs-yycy/src/main/java/cn/com/victorysoft/pcs/zbxt/zbjlgl/controller/ZbjlglController.java`
- 前端页面
  - `gtpcs-frontend/src/views/yycy/scyxh/schmrhb.vue`
  - `gtpcs-frontend/src/views/yycy/scyxh/schsjsb.vue`
  - `gtpcs-frontend/src/views/yycy/zbxt/zbjlgl/index.vue`

### 从代码上能看出的核心动作

- `ZbycscServiceImpl` 会做随机抽查，但不是纯随机：
  - 同一单位三天内不重复
  - 当月尽量覆盖未抽查单位
  - 项目 / 检查点 / 小场景也尽量避免连续重复
- `ZbycNotifyTask` 会读取夜查变更日志，找出对应记录，再按当天值班表查接收人，最后生成消息并推送。
- `ZbycChangeNotifyListener` 使用数据库 `LISTEN` 监听通知频道，再做防抖处理后触发消息处理。
- `ZbycTaskMapper.xml` 里会自动创建 PostgreSQL / KingBase 触发函数与触发器，通过 `pg_notify` 推送变更事件。
- `ZbjlglController` 则覆盖值班记录分页、保存、删除、导出，是这条业务线的日常操作面。

### 这条线体现的后端能力

- 业务规则建模能力
- 流程型功能开发能力
- 数据库触发器 / 通知机制使用能力
- 消息闭环与状态更新能力

### 最值得写进简历的内容

- 不是简单“做了值班模块”，而是做了“业务随机规则 + 变更监听 + 自动通知闭环”
- 做过数据库触发器 + `LISTEN / NOTIFY`
- 做过按排班人员自动找接收人并生成消息

### 面试时怎么讲

这条线最适合当亮点讲：

1. 先讲夜查业务不是纯表单，而是有随机策略。
2. 再讲变更后不靠人工刷新，而是数据库通知驱动自动推送。
3. 最后讲你还做了自检接口、手动处理入口，便于排障。
