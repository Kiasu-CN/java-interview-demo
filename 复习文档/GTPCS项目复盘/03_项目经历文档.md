# GTPCS 项目经历文档

## 1. 项目基础信息

- 项目名称：油气集输生产运营平台（GTPCS）
- 项目类型：生产运营类业务平台
- 项目角色：后端开发
- 后端技术：Spring Boot、MyBatis、PageHelper、定时任务、PostgreSQL / KingBase / Oracle / Taos
- 前端技术：Vue3、Element Plus、ECharts、ExcelJS
- 个人更适合强调的标签：`多模块后端开发`、`定时任务`、`规则 SQL`、`消息通知链路`、`报表与导出`

## 2. 30 秒项目介绍版

GTPCS 是一套面向油气集输生产运营场景的平台，核心覆盖生产数据同步、异常检查、消息推送、报表看板和值班夜查等业务。我在项目中主要负责后端开发，做过定时任务与数据同步、异常治理链路、消息通知链路以及报表接口开发，比较偏工程化后端，而不只是普通 CRUD。

## 3. 1 分钟项目介绍版

GTPCS 是一个面向油气集输业务的生产运营平台，既要承接生产数据，也要支撑运营管理。后端主要分成业务模块 `gtpcs-yycy` 和任务模块 `gtpcs-yycy-job`。我这段经历里，做得比较深的有几条主线：一条是基于定时任务和规则 SQL 做异常检查与质量考核，把多类异常统一落表并支撑明细查询、评分和人工补录；一条是做储气库和设备数据的同步、入库和汇总，涉及外部接口分页拉取、分批写入和班次汇总；还有一条是消息通知链路，包含排班提醒、原油零销计划变更通知以及石化通推送。此外我还参与了报表、晨报、值班夜查和导出能力建设。整体上，这个项目最能体现的是我做过比较完整的数据处理和业务闭环开发。

## 4. 简历长版项目经历

在油气集输生产运营平台（GTPCS）项目中，围绕生产数据治理与业务运营场景，参与后端多个核心模块开发与优化，负责从定时任务、SQL/Mapper、业务接口到前后端联调的完整交付。基于 `gtpcs-yycy-job` 实现多类生产数据同步任务，完成储气库采气井、注气井、泵运行记录、天然气流量计、站库设备状态等数据的分页拉取、解析清洗和分批入库，并处理 KingBase 参数数量限制下的批量写入问题；基于异常检查任务和规则型 SQL，建设数据质量监控与异常治理链路，将未审核、未按时录入、值班记录缺失、报警处置率等多类异常统一落表，再通过 `gtpcs-yycy/zlkh` 模块支撑明细查询、单位评分和人工补录。参与消息中心与石化通通知链路建设，通过动态任务调度、推送状态管理和按人独立推送策略，支撑排班提醒、生产动态和原油零销计划变更通知；同时参与报表、晨报与值班夜查模块开发，完成报表查询口径整理、导出能力联动，以及值班夜查变更通知闭环。

- 参与开发 -> 负责 / 深度参与 / 承担后端实现
- 写了接口 -> 完成接口设计与后端实现
- 做了定时任务 -> 负责定时任务与批处理链路开发
- 做了报表 -> 完成报表查询口径整理与接口开发
- 做了消息推送 -> 参与消息落库、逐人推送与状态更新闭环

## 适合替换“技术动作”

- 查询数据库 -> 编写规则 SQL / 聚合查询 / 多表关联查询
- 保存数据 -> 分批入库 / 幂等落表 / 批量写入
- 调接口 -> 对接外部数据接口并完成分页拉取与清洗
- 发通知 -> 生成消息记录并调用外部推送接口完成触达

## 适合替换“项目亮点”

- 不只是完成功能开发，还处理了数据库参数限制下的批量写入问题
- 不只是生成异常记录，还支撑了后续的评分统计和人工补录
- 不只是定时发消息，还实现了逐人推送和推送状态管理
- 不只是做报表页面，还整理了统计口径和导出能力

## 一定保留

- 定时任务 / 数据同步
- 异常检查 / 规则 SQL
- 消息推送 / 石化通
- 报表 / 晨报 / 导出

- 值班夜查自动通知
- 数据库 LISTEN / NOTIFY

## 不建议塞太多

- 过细的业务名词
- 每个报表名字都列一遍
- 每个表名都写进简历

简历上要保留的是：

**场景 + 动作 + 能力**

而不是：

**模块名大堆砌**
