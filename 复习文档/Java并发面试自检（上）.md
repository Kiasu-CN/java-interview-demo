# Java并发面试自检（上）

## 1. 什么是线程和进程？它们的关系、区别、优缺点

### 标准回答

进程是操作系统进行资源分配的基本单位，线程是 CPU 调度的基本单位。  
一个进程可以包含多个线程，线程共享进程的内存和资源，所以线程通信更方便、创建和切换成本更低；而进程之间资源隔离更强，稳定性和安全性更好，但创建、切换和通信成本更高。

### 关系

- 线程必须依附于进程存在。
- 一个进程至少有一个线程。
- 一个进程可以有多个线程并发执行。
- 同一进程内线程共享资源，不同进程之间默认隔离。

### 区别

- 进程是资源分配的基本单位，线程是调度执行的基本单位。
- 进程有独立地址空间，线程共享所属进程的地址空间。
- 进程创建和切换开销更大，线程更轻量。
- 进程间通信更复杂，线程间通信更方便。
- 进程稳定性更好，一个进程挂了通常不直接影响别的进程；线程出问题可能影响整个进程。

### 优缺点

进程优点：

- 隔离性好
- 稳定性高
- 安全性更强

进程缺点：

- 创建销毁慢
- 切换成本高
- 通信复杂

线程优点：

- 更轻量
- 并发效率高
- 共享内存，通信方便

线程缺点：

- 容易出现线程安全问题
- 容易发生死锁、竞态条件、可见性问题
- 一个线程异常可能影响整个进程

### Java 面试里怎么落地

Java 程序启动后，通常对应一个 JVM 进程。  
这个 JVM 进程里会有多个线程，比如主线程、GC 线程、线程池里的工作线程。  
所以 Java 并发里我们平时讨论的，大多数是多线程，不是多进程。

### 高频追问一句话版

1. 为什么线程切换更轻量？  
因为同一进程内线程共享大部分资源，不需要像进程那样切换完整地址空间。

2. 线程共享哪些资源？  
共享堆、方法区、文件句柄等；私有程序计数器、虚拟机栈、本地方法栈。

3. 多线程一定更快吗？  
不一定，线程太多会带来上下文切换和锁竞争开销。

### 60 秒背诵版

进程是资源分配的基本单位，线程是 CPU 调度的基本单位。一个进程可以包含多个线程，线程共享进程资源，所以线程更轻量、通信更方便，也更适合并发执行；但正因为共享资源，也更容易出现线程安全问题。进程之间隔离更强，稳定性和安全性更好，不过创建、切换和通信成本更高。Java 里启动一个应用通常就是一个 JVM 进程，而我们平时写的 Thread 和线程池，本质上都是这个进程里的多个线程。
