## 一、Java基础

### Q1：`==` 和 `equals` 区别？
1. `==` 比较基本类型的值、引用类型的地址。
2. `equals` 比较对象逻辑相等（可重写）。
3. 业务里对象判等一般依赖 `equals`。

### Q2：为什么重写 `equals` 必须重写 `hashCode`？
1. 哈希容器先按 `hashCode` 分桶，再用 `equals` 判等。
2. 若 `equals` 相等但 `hashCode` 不同，会查找/去重异常。

### Q3：`String`、`StringBuilder`、`StringBuffer` 区别？
1. `String` 不可变。
2. `StringBuilder` 可变、非线程安全、性能高。
3. `StringBuffer` 可变、线程安全、性能略低。

### Q4：自动装箱拆箱有什么坑？
1. 包装类型可为 `null`，拆箱可能 NPE。
2. `Integer` 缓存区间（-128~127）导致 `==` 结果易误判。

### Q5：重载和重写区别？
1. 重载：同名不同参数，编译期多态。
2. 重写：子类重写父类方法，运行期多态。

## 二、集合框架

### Q6：ArrayList 和 LinkedList 区别？
1. 数组 vs 双向链表。
2. 随机访问 ArrayList O(1)；LinkedList O(n)。
3. 大多数场景 ArrayList 更常用。

### Q7：ArrayList 扩容机制？
1. 容量不够触发扩容。
2. JDK8 常见为 1.5 倍扩容。
3. 扩容涉及数组复制，有成本。

### Q8：HashMap 底层结构？
1. JDK8：数组 + 链表 + 红黑树。
2. 冲突链过长可能树化。

### Q9：HashMap `put` 流程？
1. hash 扰动后定位桶。
2. 桶空直接放；桶非空则链表/树中查找。
3. key 已存在则覆盖，否则插入。
4. 超阈值可能扩容。

### Q10：为什么容量常是 2 的幂？
1. `(n - 1) & hash` 可高效定位桶。
2. 分布更均匀，减少冲突。

### Q11：HashSet 去重原理？
1. 基于 HashMap 的 key。
2. 依赖 `hashCode + equals`。

### Q12：LinkedHashMap 有什么用？
1. 维护插入/访问顺序。
2. 可用于 LRU 缓存实现。

### Q13：TreeMap/TreeSet 特点？
1. 红黑树，有序。
2. 支持范围查询，复杂度 O(log n)。

### Q14：什么是 fail-fast？
1. 迭代期间结构被并发修改，快速失败。
2. 常见异常 `ConcurrentModificationException`。

### Q15：增强 for 中删除为什么报错？
1. 本质使用 Iterator。
2. 直接调用集合 remove 破坏 expectedModCount。
3. 应改为 `Iterator.remove()`。

## 三、并发编程

### Q16：线程创建方式有哪些？
1. Thread。
2. Runnable。
3. Callable + Future。
4. 线程池（推荐）。

### Q17：`synchronized` 和 `ReentrantLock` 区别？
1. 前者语法级，后者 API 级。
2. Lock 支持可中断、超时、公平锁等高级能力。

### Q18：`volatile` 能保证什么？
1. 可见性。
2. 有序性（部分重排约束）。
3. 不保证复合操作原子性。

### Q19：CAS 是什么？有什么问题？
1. Compare-And-Swap，无锁原子更新。
2. 问题：ABA、自旋开销。
3. 常见解法：版本号（AtomicStampedReference）。

### Q20：AQS 是什么？
1. 同步器框架，维护 state + FIFO 队列。
2. ReentrantLock、Semaphore 等基于它实现。

### Q21：线程池核心参数？
1. corePoolSize、maxPoolSize。
2. keepAliveTime、workQueue。
3. threadFactory、RejectedExecutionHandler。

### Q22：拒绝策略有哪些？
1. AbortPolicy（抛异常）。
2. CallerRunsPolicy（调用者执行）。
3. DiscardPolicy（丢弃）。
4. DiscardOldestPolicy（丢最老）。

### Q23：为什么不推荐 Executors 默认工厂？
1. 队列/线程上限不受控。
2. 容易引发 OOM 或过载。

### Q24：ConcurrentHashMap 为什么线程安全？
1. JDK8 CAS + synchronized（桶级）。
2. 读写并发性能优于全局锁。

### Q25：ThreadLocal 原理与风险？
1. 每线程独立变量副本。
2. key 弱引用，value 可能残留导致泄漏。
3. 用完要 remove。

## 四、JVM

### Q26：JVM 运行时内存区域？
1. 堆。
2. 虚拟机栈。
3. 方法区。
4. 程序计数器。
5. 本地方法栈。

### Q27：对象如何判断可回收？
1. 可达性分析，从 GC Roots 出发。
2. 不可达对象可回收。

### Q28：常见 GC 算法？
1. 标记-清除。
2. 复制。
3. 标记-整理。

### Q29：类加载过程？
1. 加载、验证、准备、解析、初始化。

### Q30：双亲委派好处？
1. 避免重复加载。
2. 防核心类被篡改。

### Q31：常见 OOM 类型？
1. Java heap space。
2. Metaspace。
3. StackOverflowError。

### Q32：Full GC 频繁怎么排查？
1. 监控观察频率与停顿。
2. 看 GC 日志。
3. dump 分析对象占用。
4. 回到代码与缓存策略定位根因。

## 五、Spring / Spring Boot / MVC

### Q33：IOC 是什么？
1. 对象创建和依赖关系交由容器管理。

### Q34：AOP 作用？
1. 把日志、事务、鉴权等横切逻辑解耦。

### Q35：Bean 生命周期？
1. 实例化 -> 属性注入 -> 初始化回调 -> 使用 -> 销毁。

### Q36：事务传播行为常用哪些？
1. REQUIRED（默认）。
2. REQUIRES_NEW。
3. SUPPORTS。

### Q37：事务失效常见场景？
1. 同类自调用。
2. 异常被吞。
3. 非 public 方法或代理未生效。

### Q38：Spring Boot 自动配置原理？
1. 条件装配 + 自动配置类导入。
2. 按 classpath/配置决定生效。

### Q39：拦截器、过滤器、AOP 区别？
1. 过滤器偏 Servlet 层。
2. 拦截器偏 Spring MVC 请求链。
3. AOP 面向方法切面。

### Q40：参数校验如何做？
1. `@Valid/@Validated`。
2. BindingResult 或全局异常统一处理。

## 六、MyBatis

### Q41：`#{}` 和 `${}` 区别？
1. `#{}` 预编译参数，防注入。
2. `${}` 字符串拼接，有注入风险。

### Q42：MyBatis 执行流程？
1. Mapper 代理。
2. SQL 解析与参数绑定。
3. Executor 执行，映射结果返回。

### Q43：一级缓存和二级缓存区别？
1. 一级缓存：SqlSession 级别。
2. 二级缓存：Mapper 级别。

## 七、MySQL

### Q44：为什么索引用 B+树？
1. 磁盘 IO 友好，树高低。
2. 叶子有序，范围查询高效。

### Q45：聚簇索引和二级索引区别？
1. 聚簇索引叶子存整行数据。
2. 二级索引叶子存主键值，可能回表。

### Q46：最左前缀原则是什么？
1. 联合索引从最左列开始匹配。
2. 跳过前导列会导致索引利用变差。

### Q47：索引失效常见场景？
1. 对索引列做函数/隐式类型转换。
2. 模糊查询前置 `%`。
3. 低选择性错误使用。

### Q48：事务隔离级别有哪些？
1. RU、RC、RR、Serializable。
2. 隔离越高并发通常越低。

### Q49：MVCC 是什么？
1. 多版本并发控制。
2. 通过版本链 + ReadView 实现快照读。

### Q50：行锁、间隙锁、Next-Key 锁区别？
1. 行锁锁记录。
2. 间隙锁锁区间。
3. Next-Key = 行锁 + 间隙锁。

### Q51：Explain 怎么看？
1. `type`、`key`、`rows`、`extra`。
2. 目标：减少扫描行数，尽量索引命中。

### Q52：慢SQL优化步骤？
1. 定位慢SQL。
2. Explain 分析。
3. 索引/SQL重写。
4. 回归验证与监控。

## 八、Redis

### Q53：Redis 常见数据结构场景？
1. String：缓存对象/计数。
2. Hash：对象属性。
3. List：队列。
4. Set：去重。
5. ZSet：排行榜。

### Q54：穿透、击穿、雪崩怎么区分？
1. 穿透：不存在key。
2. 击穿：热点key失效瞬时并发。
3. 雪崩：大量key同一时刻失效。

### Q55：对应解决方案？
1. 穿透：布隆过滤器/空值缓存。
2. 击穿：互斥锁/逻辑过期。
3. 雪崩：过期打散/多级缓存/限流。

### Q56：RDB 和 AOF 区别？
1. RDB 恢复快，可能丢秒级数据。
2. AOF 数据更完整，文件更大、恢复慢。

### Q57：分布式锁要注意什么？
1. 原子加锁（SET NX EX）。
2. 释放锁防误删（校验标识+Lua）。
3. 续期与超时处理。

## 九、网络与安全

### Q58：HTTP 和 HTTPS 区别？
1. HTTPS = HTTP + TLS，加密、认证、防篡改。

### Q59：TCP 三次握手意义？
1. 双方确认收发能力与初始序列号。

### Q60：Cookie、Session、JWT 区别？
1. Cookie 存客户端。
2. Session 存服务端。
3. JWT 无状态，扩展性好但撤销复杂。

### Q61：XSS、CSRF、SQL注入如何防？
1. XSS：输出转义、CSP。
2. CSRF：token + SameSite。
3. SQL注入：预编译参数。

## 十、项目场景题（高频）

### Q62：如何做接口幂等？
1. 幂等 token。
2. 唯一索引。
3. 状态机防重复处理。

### Q63：高并发下如何防超卖？
1. 原子扣减库存。
2. 分布式锁/乐观锁。
3. 异步削峰与最终一致性。

### Q64：缓存一致性怎么做？
1. 旁路缓存（更新DB后删缓存）。
2. 延时双删/重试补偿。
3. 关键路径加消息同步。

### Q65：线上故障排查思路？
1. 先止血（限流/降级）。
2. 再定位（日志、监控、链路）。
3. 后修复（回滚/补偿）。
4. 最后复盘（预防机制）。
