# 数据结构补强全攻略

目标：不是重学一遍教科书式数据结构，而是补齐 Java 后端面试和 LeetCode 高频刷题所需要的核心直觉、常见操作、典型题型和易错点。

适用人群：

- 已经开始刷题，但总觉得“题会看，不会选结构”
- 栈、队列、链表、哈希、树这些题做起来不够稳
- 代码能写，但边界、语义、适用场景容易混
- 想补的是“会做题、会面试表达”的数据结构能力

---

## 一、先建立一个总认识

数据结构不是背定义，而是回答这 4 个问题：

1. 数据怎么存
2. 数据怎么取
3. 哪种操作快
4. 遇到什么题应该先想到它

你现在真正要补的，不是“百科式知识点”，而是这三层：

- 概念层：它是什么，适合干什么
- 编码层：Java 里怎么写，常用 API 是什么
- 题感层：看到什么题型会想到它

---

## 二、你最该优先补的 6 大结构

### 1. 数组 Array

最核心特点：

- 连续内存
- 支持随机访问
- 通过下标取值很快
- 中间插入、删除代价高

你必须形成的直觉：

- 只要题目频繁按下标访问，就先想到数组
- 只要题目强调“有序”，通常数组 + 二分会出现
- 只要题目要求原地修改，数组题很常见

常见题型：

- 二分查找
- 双指针
- 滑动窗口
- 前缀和
- 原地去重、原地移动

Java 常用写法：

```java
int[] nums = new int[10];
int n = nums.length;
```

容易混的点：

- 下标从 `0` 开始
- `length` 是属性，不是方法
- 边界最容易错在 `0` 和 `length - 1`

---

### 2. 链表 LinkedList / ListNode

最核心特点：

- 节点通过指针连接
- 不支持随机访问
- 插入、删除某个已知节点附近通常更灵活

你必须形成的直觉：

- 链表题本质上大多是“指针操作题”
- 真正难点不是值，而是引用关系
- 画图比硬想更有效

常见题型：

- 反转链表
- 合并有序链表
- 删除倒数第 N 个节点
- 判断环
- 找中点

Java 中刷题常见定义：

```java
class ListNode {
    int val;
    ListNode next;
    ListNode() {}
    ListNode(int val) { this.val = val; }
    ListNode(int val, ListNode next) {
        this.val = val;
        this.next = next;
    }
}
```

必须掌握的套路：

- 虚拟头节点 `dummy`
- 快慢指针
- 前驱节点 `prev`
- 反转三件套：`prev`、`curr`、`next`

容易混的点：

- 删除节点时，改的是 `prev.next`
- 遍历结束条件经常写错
- `head == null` 和 `head.next == null` 要分开看

---

### 3. 栈 Stack

最核心特点：

- 后进先出，`LIFO`
- 只能从一端进出

你必须形成的直觉：

- 栈特别适合处理“最近的、最后压进去的东西”
- 只要涉及括号匹配、撤销、表达式、单调性，就要想到栈

常见题型：

- 有效的括号
- 最小栈
- 字符串解码
- 单调栈题

Java 里建议：

- 优先用 `Deque` 的实现类 `ArrayDeque`
- 不建议在新代码里优先使用老的 `Stack`

常见写法：

```java
Deque<Integer> stack = new ArrayDeque<>();
stack.push(1);
int top = stack.pop();
int peek = stack.peek();
boolean empty = stack.isEmpty();
```

容易混的点：

- `push/pop/peek` 是按“栈语义”在用
- `Deque` 既能当栈，也能当队列

---

### 4. 队列 Queue

最核心特点：

- 先进先出，`FIFO`
- 队尾进入，队头出去

你必须形成的直觉：

- 谁先来谁先处理，就想到队列
- 分层、逐层扩散、排队处理场景常见

常见题型：

- 用栈实现队列
- 二叉树层序遍历
- BFS 广度优先搜索

Java 常见写法：

```java
Queue<Integer> queue = new LinkedList<>();
queue.offer(1);
int front = queue.poll();
int peek = queue.peek();
boolean empty = queue.isEmpty();
```

更常见的工程刷题写法：

```java
Deque<Integer> queue = new ArrayDeque<>();
queue.offerLast(1);
int front = queue.pollFirst();
int peek = queue.peekFirst();
```

容易混的点：

- `offer` 是入队
- `poll` 是出队
- `peek` 是看队头但不删除

---

### 5. 双端队列 Deque

最核心特点：

- 头尾都可以进出
- 能模拟栈，也能模拟队列

你必须形成的直觉：

- Java 刷题里 `Deque` 很重要
- 很多时候不用纠结“到底是栈还是队列”，先看是否需要双端操作

高频应用：

- 模拟栈
- 模拟队列
- 单调队列
- 滑动窗口最大值

Java 常见写法：

```java
Deque<Integer> deque = new ArrayDeque<>();
deque.offerFirst(1);
deque.offerLast(2);
int a = deque.pollFirst();
int b = deque.pollLast();
```

容易混的点：

- `First` 对应头部
- `Last` 对应尾部
- 把“从头出”还是“从尾出”想清楚再写

---

### 6. 哈希表 HashMap / HashSet

最核心特点：

- 通过键快速定位值
- 平均查询效率高

你必须形成的直觉：

- 只要题目需要“快速查找某个值是否存在”，先想哈希
- 只要题目需要“计数、去重、映射关系”，先想哈希

常见题型：

- 两数之和
- 有效的字母异位词
- 最长连续序列
- 高频统计

Java 常见写法：

```java
Map<Integer, Integer> map = new HashMap<>();
map.put(1, 10);
int value = map.get(1);
boolean exists = map.containsKey(1);
```

```java
Set<Integer> set = new HashSet<>();
set.add(1);
boolean exists = set.contains(1);
```

容易混的点：

- `HashMap` 是键值对
- `HashSet` 本质上更偏“只关心有没有”
- `containsKey` 和 `containsValue` 不要混

---

## 三、再往上走：必须补的 3 个结构

### 1. 树，尤其是二叉树

为什么必须补：

- 面试高频
- 递归思想的重要载体
- 许多搜索、遍历、分治思想都在树上体现得最明显

你必须掌握：

- 前序、中序、后序遍历
- 层序遍历
- 递归返回值设计
- 二叉搜索树的性质

高频题：

- 最大深度
- 对称二叉树
- 翻转二叉树
- 层序遍历
- 验证二叉搜索树

---

### 2. 堆 / 优先队列 PriorityQueue

为什么必须补：

- TopK 题太常见
- 动态维护最值时特别高频

Java 常见写法：

```java
PriorityQueue<Integer> minHeap = new PriorityQueue<>();
PriorityQueue<Integer> maxHeap = new PriorityQueue<>((a, b) -> b - a);
```

高频题：

- 数组中的第 K 个最大元素
- 前 K 个高频元素
- 合并 K 个有序链表

---

### 3. 字符串

虽然很多人不把它单独当数据结构，但对刷题非常重要。

你必须掌握：

- `String`
- `StringBuilder`
- 字符数组
- 双指针处理字符串
- 字符统计

高频题：

- 最长公共前缀
- 验证回文串
- 无重复字符的最长子串
- 最小覆盖子串

---

## 四、Java 刷题里最该熟的容器组合

这一块很重要，因为你不只是要懂概念，还要会在 Java 里立刻写出来。

### 1. 栈

推荐：

```java
Deque<Integer> stack = new ArrayDeque<>();
```

### 2. 普通队列

推荐：

```java
Queue<Integer> queue = new LinkedList<>();
```

或者：

```java
Deque<Integer> queue = new ArrayDeque<>();
```

### 3. 双端队列

推荐：

```java
Deque<Integer> deque = new ArrayDeque<>();
```

### 4. 哈希映射

推荐：

```java
Map<Character, Integer> map = new HashMap<>();
```

### 5. 集合去重

推荐：

```java
Set<Integer> set = new HashSet<>();
```

### 6. 小根堆 / 大根堆

推荐：

```java
PriorityQueue<Integer> minHeap = new PriorityQueue<>();
PriorityQueue<Integer> maxHeap = new PriorityQueue<>((a, b) -> b - a);
```

## 五、刷题时怎么判断该用什么结构

看到题目时，可以先问这几个问题：

### 1. 需要快速判断“有没有”吗

- 是：优先想 `HashSet` / `HashMap`

### 2. 需要按下标随机访问吗

- 是：优先想数组

### 3. 需要维护“最近进入”的元素吗

- 是：优先想栈

### 4. 需要维护“最早进入”的元素吗

- 是：优先想队列

### 5. 需要头尾都操作吗

- 是：优先想双端队列

### 6. 需要动态维护最大/最小或 TopK 吗

- 是：优先想堆 / 优先队列

### 7. 题目本身就是节点连接关系吗

- 是：优先想链表或树

## 六、你现在最容易薄弱的知识点

结合常见刷题卡点，很多人会卡在以下几类意识上：

### 1. 只会背定义，不会映射题型

表现：

- 知道栈是 `LIFO`
- 但看到括号匹配时想不到用栈

补法：

- 每学一个结构，至少配 2 到 4 道代表题

### 2. 概念懂了，但 Java 写法不熟

表现：

- 知道队列先进先出
- 但不知道 Java 里用什么类最顺手

补法：

- 每个结构都手写 3 次基础 API

### 3. 知道用什么结构，但不会分析边界

表现：

- 栈、队列选对了
- 但空结构、单元素、最后一次出队时容易错

补法：

- 配合 [算法题自检清单.md](d:\code\vscode\demo\复习文档\算法题自检清单.md) 一起练

### 4. 把抽象结构和具体类混在一起

表现：

- 把“栈”和 `Stack` 类画等号
- 把“队列”和 `LinkedList` 画等号

要意识到：

- 栈、队列、双端队列是抽象结构
- `ArrayDeque`、`LinkedList`、`PriorityQueue` 是 Java 里的具体实现/容器

## 七、最适合你的补强方式

不要按教材式重学，建议按这个顺序补。

### 第一步：补结构定义和使用场景

目标：

- 一看到题，能初步判断该用什么结构

方法：

- 每个结构只写 3 个东西
- 它的核心特征
- 它适合什么题
- 它最常见的坑是什么

### 第二步：补 Java API 落地

目标：

- 不再停在“我知道思路，但不会写容器”

方法：

- 每个容器手写一遍增删查取空

例如：

```java
Deque<Integer> stack = new ArrayDeque<>();
stack.push(1);
stack.pop();
stack.peek();
stack.isEmpty();
```

### 第三步：补题型映射

目标：

- 看到题目特征就能条件反射想到结构

方法：

- 一个结构至少绑定 2 到 4 道高频题

例如：

- 栈：`LC20`、`LC155`
- 队列：`LC232`、层序遍历
- 双端队列：`LC239`
- 哈希：`LC1`、`LC128`
- 链表：`LC206`、`LC141`

### 第四步：补边界意识

目标：

- 不只是会写，还能写对

方法：

- 重点检查空结构、单元素、最后一次操作

## 九、补强时最容易走偏的地方

### 1. 只看概念，不做题

结果：

- 理论上都懂，题一做就卡

### 2. 只做题，不回头总结结构

结果：

- 做过一道会一道，迁移能力差

### 3. 只背 Java API，不理解结构本质

结果：

- 容器会写，但不知道为什么题目要这么设计

### 4. 一上来学太重的结构

结果：

- 图、并查集、线段树学了很多
- 但基础的栈队列链表反而不稳

## 十、你现在最应该建立的能力

不是“我背了多少定义”，而是下面这 4 种能力：

1. 看到题目，能判断该用什么结构
2. 知道这个结构在 Java 里怎么写
3. 知道这个结构最容易错在哪
4. 能把一道题归纳回结构本身

---

## 十一、面试视角下，补数据结构到底在补什么

面试官通常不是想听你背定义，而是想看你：

- 有没有结构意识
- 能不能做出合适的数据结构选择
- 是否理解时间复杂度和空间复杂度
- 写代码时边界是否稳

所以你补数据结构，最终目标不是“像教材一样会讲”，而是：

- 做题能选对
- 写代码能写稳
- 面试能讲清楚

---

## 十二、最后给你的结论

你现在确实应该补数据结构，但不用重学全套理论。

你最该补的是：

- 高频结构的本质
- Java 容器落地写法
- 题型和结构之间的映射关系
- 常见边界错误

一句话总结：

**你要补的不是“知识点数量”，而是“结构直觉 + Java 落地 + 题型映射”。**
